## 1. 문제 (Problems)

> 고객이 실제로 겪는 불편함

**핵심 문제 3가지**

1. **분산된 커뮤니케이션**: 소규모 모임 주최자가 공지·참석 확인·카풀 조율·비용 정산을 카카오톡, 구글 폼, 엑셀 등 여러 도구에 걸쳐 따로따로 처리해야 함
2. **참석자 집계 비효율**: 메시지로 흩어진 참석 의사를 주최자가 일일이 수동으로 집계해야 하며, 변경 시 재집계 부담 발생
3. **정산의 어려움**: 모임 후 누가 얼마를 내야 하는지, 납부 완료 여부를 추적하는 수단이 없어 정산이 감정 소모로 이어짐

### 기존 대안 (Existing Alternatives)

| 대안 | 한계 |
|------|------|
| 카카오톡 단체채팅방 | 정보가 메시지에 묻혀 공지·참석·정산이 모두 흩어짐 |
| 네이버 밴드 | 기능 과잉, 소규모 정기 모임에 불필요하게 무거운 UI |
| 구글 폼 | 일회성 설문에 그침, 카풀·정산 통합 불가 |
| 엑셀 / 노션 수동 관리 | 주최자 단독 작업, 멤버 자율 참여 불가 |

## 2. 해결안 (Solutions)

> 문제를 해결하는 우리 서비스의 핵심 기능

1. **초대 코드 기반 그룹 생성·가입** — 8자리 코드로 폐쇄형 그룹을 빠르게 구성, 주최자가 멤버를 손쉽게 관리
2. **이벤트 RSVP** — 참석 / 미정 / 불참을 클릭 한 번으로 선택, 주최자는 실시간 참석자 현황 파악
3. **공지 게시 + 핀 고정** — 준비물·장소 변경 등 중요 공지를 그룹·이벤트 단위로 즉시 전파
4. **카풀 조율** — 운전자 빈 좌석 등록 + 동승 희망 등록으로 이동 편의 조율
5. **균등 분할 정산 + 납부 추적** — 총액 입력 후 1인당 금액 자동 계산, 납부 여부를 체크박스로 추적

### 핵심 지표 (Key Metrics)

| 지표 | 정의 | 목표 (MVP) |
|------|------|-----------|
| 월간 활성 그룹 수 (MAG) | 한 달 내 이벤트가 1회 이상 생성된 그룹 수 | 50개 |
| 이벤트당 RSVP 응답률 | 이벤트 멤버 대비 RSVP 제출 비율 | 70% 이상 |
| 정산 완료율 | 생성된 정산 중 모든 멤버 납부 완료 비율 | 60% 이상 |
| 주최자 재사용률 (4주) | 그룹 생성 후 4주 내 이벤트 2회 이상 생성 비율 | 40% 이상 |

## 3. 고유의 가치 제안 (Unique Value Proposition)

> 고객이 왜 이 서비스를 써야 하는지 한 문장으로

**"소규모 정기 모임을 위한 올인원 관리 도구 — 공지·참석·카풀·정산을 한 곳에서"**

- 흩어진 채팅을 정리할 필요 없이 **모임의 모든 것**을 하나의 서비스에서 처리
- 주최자의 반복 노동(집계·알림·정산)을 **자동화**
- 멤버는 초대 코드로 즉시 참여 — **별도 가입 과정 최소화**

---

## 4. 경쟁우위 (Unfair Advantage)

> 쉽게 복사하거나 구매할 수 없는 독창적인 강점

1. **기능 조합의 특수성**: 공지 + RSVP + 카풀 + 정산을 소규모 모임 맥락에 최적화한 조합은 기존 어떤 서비스도 제공하지 않음
2. **폐쇄형 초대 코드 모델**: 공개 검색 없이 코드로만 입장 → 소규모 사적 모임의 프라이버시 요구에 부합
3. **카풀 조율 기능**: 이동 관련 기능을 모임 관리 서비스 안에 내장한 사례가 거의 없음 (차별점)
4. **주최자 중심 설계**: 모임 운영의 고통 포인트를 가장 깊이 이해한 도메인 지식

### 채널 (Channel)

> 서비스가 고객과 만나는 경로

- **바이럴 초대**: 멤버가 초대 코드를 공유하며 자연스럽게 확산 (K-factor 기대)
- **커뮤니티 마케팅**: 수영·헬스·러닝 관련 네이버 카페, 오픈채팅방 직접 홍보
- **SEO**: "동호회 관리 앱", "모임 정산 앱" 등 검색 최적화
- **입소문**: 첫 주최자 경험 개선 → 멤버에게 자발적 추천 유도

## 5. 고객군 (Customer Segments)

> 우리 서비스가 해결하는 문제를 가진 구체적인 사람들

**1차 타겟 (주최자)**
- 20~40대, 수영·헬스·러닝 등 소규모 정기 운동 모임 주최자
- 주 1~2회 정도 정기적으로 모임을 운영하며 참석자 관리에 피로감을 느끼는 사람

**2차 타겟**
- 스터디 그룹·독서 모임 주최자 (정기성 + 참석 확인 니즈 동일)
- 동창회·친목 소모임 운영자

**3차 타겟 (멤버)**
- 위 그룹에 속한 참여 멤버 — 주최자의 RSVP 요청에 응하며 서비스를 경험

### 초기 사용자 (Early Adopters)

> 가장 먼저 채택할 가능성이 높은 고객

- 현재 카카오톡으로 **10~30명 규모** 정기 운동 모임을 운영하는 주최자
- 매 모임마다 참석 확인과 비용 정산을 동시에 처리해야 하는 불편함을 **이미 인지**하고 있는 사람
- 새로운 앱 도입에 거부감이 낮고 효율화에 관심 있는 30대 직장인 주최자
